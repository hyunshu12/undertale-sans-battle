> 제작: 프로그래밍 수행평가 (한국디지털미디어고등학교, 2026)

## 1. 게임 소개

언더테일(Undertale)의 최종 보스 **샌즈(Sans)와의 전투**를 그대로 옮긴 턴제 탄막 회피 게임입니다.
원작 웹 버전인 **Bad Time Simulator(BTS)**의 공격 데이터(CSV 24종)·물리·연출을 1:1로 재현했습니다.

- 플레이어는 빨간 하트(영혼)를 조종해 샌즈의 탄막을 피합니다.
- 외부 라이브러리·설치 없이 **Visual Studio에서 F5만 누르면** 바로 실행됩니다.
- 모든 대사·UI는 한국어(픽셀 폰트: 네오둥근모).

---

## 2. 실행 방법

1. `UndertaleSans/UndertaleSans.sln` 을 Visual Studio로 엽니다.
2. **F5**(디버그 실행) 또는 **Ctrl+F5**(빌드 후 실행)를 누릅니다.
3. 게임 창이 전체화면으로 바로 실행됩니다.

> VS가 "프로젝트 다시 대상 지정"을 물으면 **확인**을 누르세요(빌드 영향 없음).
> IntelliSense가 `windows.h`에 빨간 줄을 그어도 실제 빌드(F5)는 정상입니다.

| 키 | 동작 |

| **ESC** | 종료 |

## 4. 게임 목표와 전체 흐름

**목표:** 샌즈의 모든 공격을 견뎌내고 **특수 공격까지 생존**하면 승리합니다. (HP가 0이 되면 패배)

```
타이틀 화면
   │  Z
난이도 선택 (EASY / NORMAL / HARD)
   │  Z
인트로 독백 (Z로 진행)
   │
┌─→ 적 턴: 샌즈의 탄막 공격 (영혼으로 회피)
│      │  공격 종료
│   메뉴 턴: FIGHT / ACT / ITEM / MERCY 중 선택
│      │  행동 완료 → 다음 공격
└──────┘  (23턴 반복)
   │
특수 공격(sans_final) 생존 → 🏆 승리 → 타이틀
   또는 HP 0 → 💀 게임오버 → (4초 후 자동) 타이틀
```

---

## 5. 화면 구성

- **위쪽**: 샌즈(머리·몸통·다리, 표정/팔 동작이 공격에 따라 변함)와 말풍선
- **가운데**: 전투 박스(흰 테두리) — 영혼과 탄막이 이 안에서 움직임. 공격마다 크기가 변함
- **아래쪽**:
  - `CHARA  LV 19` (캐릭터 이름/레벨)
  - `HP` 라벨 + 노란 HP 바 + `92 / 92` 숫자
  - **FIGHT · ACT · ITEM · MERCY** 버튼 (선택된 버튼은 노랗게 강조 + 하트 커서)

---

## 6. 전투 시스템

### 6-1. 턴 구조
매 턴은 **메뉴에서 행동 선택 → 샌즈의 공격(적 턴)** 순서로 진행됩니다.
어떤 행동(FIGHT/ACT/ITEM/MERCY)을 골라도 그 뒤에 반드시 샌즈가 공격해 옵니다.

### 6-2. 메뉴 4가지 행동
| 버튼 | 효과 |
|---|---|
| **FIGHT** | 타겟 바가 박스를 가로질러 움직임 → **가운데 노란 타겟존에서 Z**로 멈추면 **명중!**, 빗나가면 샌즈가 회피("빗나감!"). *(샌즈는 끈질겨서 명중해도 쓰러지지 않음 — 승리는 생존으로)* |
| **ACT** | 샌즈를 관찰(Check). "공격 1 방어 1" 등 정보 표시 (진행하면 대사가 바뀜) |
| **ITEM** | 회복 아이템 사용. 좋은 것부터 소비(버터스카치 파이 +99 → … → 전설의 영웅 +40) |
| **MERCY** | 자비를 베풂. **EASY에서 5턴 이상 버티면 자비 승리** 가능 (NORMAL/HARD에선 불가) |

### 6-3. 영혼(하트) 두 가지 모드
- 🔴 **빨간 영혼**: 8방향 자유 이동 (속도 150)
- 🔵 **파란 영혼**: **중력**이 작용 — 바닥에 떨어지고, **위/W로 점프**해서 뼈를 넘습니다.
  중력 방향이 바뀌는 공격(SansSlam)도 있습니다.

### 6-4. HP와 KARMA(카르마)
- 시작 HP: **92** (피격당 1 감소, 피격 후 짧은 무적 시간)
- **KARMA(보라색)**: 피격 시 HP 바 오른쪽 끝에 보라색으로 누적됩니다.
  카르마는 시간이 지나며 **HP를 서서히 갉아먹습니다**(카르마가 많을수록 빠르게).
  → 맞고 나서 가만히 있으면 추가 피해를 입으니 주의!

### 6-5. 파란 뼈 / 흰 뼈 (중요!)
- ⚪ **흰 뼈**: 닿으면 무조건 피해
- 🔵 **파란 뼈**: **움직일 때만** 피해 → 파란 뼈가 지나갈 땐 **멈춰 있으면** 통과합니다!

## 7. 공격 패턴 가이드 (BTS 원본 순서)

샌즈는 정해진 순서로 공격합니다. (HitAttempts = 버틴 턴 수)

| 순서 | 공격 | 설명 / 회피 팁 |
|---|---|---|
| 1 | **sans_intro** | 오프닝 — 게이스터 블래스터 난사 + 뼈. "준비됐어?" 후 시작 |
| 2~4 | **bonegap / bluebone** | 좌우에서 들어오는 뼈 벽의 **빈틈**에 영혼을 맞춤. 파란 뼈 구간은 멈춰서 통과 |
| 5~10 | **platforms / platformblaster** | 파란 영혼 — 움직이는 발판 위에서 점프하며 블래스터 빔 회피 |
| 11~13 | **bonegap1fast / boneslideh** | 빨라진 뼈. 침착하게 빈틈 추적 |
| 14 | **sans_spare** | 샌즈가 잠시 쉬는 턴 |
| 15~22 | **multi / randomblaster / bonestab** | 본격 난전 — 다방향 블래스터·찌르는 뼈(경고선 보고 회피) |
| 23 | **sans_final** | **특수 공격** — 중력이 사방으로 바뀌며 벽으로 내리꽂힘. 살아남으면 **승리!** |

### 게이스터 블래스터
샌즈의 해골 대포. **충전(턱 벌림) → 흰 빔 발사**. 빔이 나올 위치를 보고 미리 비켜 있으세요.

### 찌르는 뼈(BoneStab)
**경고음(Warning)과 예고선**이 먼저 뜬 뒤 해당 방향에서 뼈가 쑥 들어옵니다. 예고를 보고 반대편으로!

## 8. 난이도

| 난이도 | 무적 시간 | 카르마 | 아이템 | 자비 |
|---|---|---|---|---|
| **EASY** | 0.4초 (넉넉) | 절반 | 8개 | **5턴 후 MERCY로 자비 승리** |
| **NORMAL** | 0.034초 (BTS 원본) | 100% | 8개 | 없음(생존만) |
| **HARD** | 0.017초 (절반) | 100% | **3개** | 없음(생존만) |

> EASY = 채점·연습용 / NORMAL = 웹 원본과 동일 / HARD = 무적시간 짧고 아이템 적음

## 9. 엔딩 (총 3종)

| 엔딩 | 조건 | 결과 |
|---|---|---|
| 💀 **게임오버** | HP가 0이 됨 | 하트가 금이 가 6조각으로 부서짐 → 4초 후 자동으로 타이틀 복귀 |
| 🏆 **최종 승리** | 23턴을 버텨 **특수 공격(sans_final)까지 생존** | 샌즈 "헉... 헉... 좋아, 인정하지... 네가 이겼어." → 타이틀 (모든 난이도의 정식 승리) |
| 🕊️ **자비 승리** | **EASY에서 5턴 후 MERCY 선택** | 샌즈 "흠... 좋아. 오늘은 이쯤에서 봐주지." → 타이틀 (EASY 전용) |

> NORMAL·HARD에서는 자비 승리가 막혀 있어 **오직 끝까지 생존**해야 이깁니다(BTS 원본과 동일).

## 10. 공략 팁

1. **파란 뼈가 보이면 멈추세요.** 움직이면 맞습니다.
2. **맞은 직후 카르마가 차면** 가만히 있지 말고 계속 회피 — 카르마가 HP를 갉아먹습니다.
3. **블래스터 빔은 위치 예측**이 핵심. 빔이 그어질 줄을 미리 피하세요.
4. **찌르는 뼈는 경고선**을 보고 반대로.
5. **좁은 틈은 Shift(정밀 이동)**로 통과.
6. 어렵다면 **EASY로 연습** 후 NORMAL에 도전.

## 11. 구현 기술 (요약)

- **언어/환경**: C (MSVC), Win32 GDI 더블 버퍼링, 외부 라이브러리 0 (gdi32/user32/winmm/msimg32만 자동 링크)
- **렌더링**: 메모리 DC에 그린 뒤 StretchBlt로 전체화면 레터박스 출력(비율 유지) — 깜빡임 없음
- **공격 엔진**: BTS의 공격 24종을 **CSV 바이트코드**로 동봉하고, 자체 제작한 **VM 인터프리터**(`vm.c`)가 매 프레임 해석해 탄막을 생성 → 원본과 패턴이 동일
- **물리**: 가변 중력 밴드(540/180/450)·점프·4px 히트박스·플랫폼 탑승 — BTS 소스와 수치 일치
- **오디오**: BGM(메가로바니아)은 PlaySound, 효과음 12종은 MCI로 동시 재생
- **폰트**: 네오둥근모(OFL) 런타임 로드(설치 불필요)
